# Bradobrei Party Frontend

Фронтенд для локальной разработки и дальнейшего развития системы `Bradobrei Party` на `React + TypeScript + Vite`.

## Что реализовано

- проксирование запросов Vite к backend для локальной разработки
- хранение JWT в `localStorage` через отдельный сервис
- единый API-клиент без хардкода адресов в коде
- структура проекта по слоям:
  - `src/pages`
  - `src/components`
  - `src/api/services`
  - `src/types/dto`
  - `src/shared`
- роутинг через `react-router-dom`
- страницы:
  - `/auth` — вход и регистрация пользователя
  - `/employees/new` — регистрация нового сотрудника
  - `/reports/employees` — отчёт 2.2.1
  - `/reports/salon-activity` — отчёт 2.2.2
  - `/reports/master-activity` — отчёт 2.2.4
- общие таблицы отчётов вынесены в `src/shared/reportTables.tsx`

## Почему не нужен отдельный чат для фронтенда

Пока отдельный чат не нужен: здесь уже накоплен контекст по DTO, ролям, защищённым маршрутам и структуре backend. Для задач, где фронтенд и backend связаны требованиями ТЗ, в одном чате даже удобнее и безопаснее.

Отдельный чат может понадобиться только если вы захотите:

- отдельно прорабатывать чистый UI/UX без опоры на текущий backend
- делать альтернативный фронтенд-эксперимент вне существующей структуры

## Настройка окружения

Параметры backend вынесены в `.env`:

```env
VITE_API_SCHEME=http
VITE_API_HOST=localhost
VITE_API_PORT=9000
VITE_API_BASE_PATH=/api/v1
```

Файл-шаблон лежит в `.env.example`.

В dev-режиме фронтенд использует относительные запросы и proxy Vite, поэтому VPN и прямой доступ к backend меньше мешают локальной работе.

## Установка

Из папки `frontend`:

```bash
npm install
```

Если зависимости уже установлены, повторять не нужно.

## Локальный запуск

Сначала поднимите backend, затем из папки `frontend`:

```bash
npm run dev
```

По умолчанию Vite откроет локальный dev server, а запросы на:

- `/api`
- `/docs`
- `/swagger`
- `/health`

будут проксироваться на backend, указанный в `.env`.

## Сборка

```bash
npm run build
```

## Архитектура фронтенда

### `src/api`

- `config.ts` — сборка API-конфига из `import.meta.env`
- `client.ts` — общий `fetch`-клиент
- `services/*` — методы по доменам

### `src/types/dto`

- описывает формы backend DTO и отчётных ответов
- можно хранить полезные фронтовые уточнения, если они не ломают контракт API

### `src/shared`

- общие форматтеры
- конфигурации таблиц отчётов
- справочные списки для селектов

### `src/components`

- `AppShell` — каркас приложения после авторизации
- `ProtectedRoute` — защита приватных маршрутов
- `DataTable` — универсальная таблица отчётов

### `src/pages`

- прикладные страницы, привязанные к реальным сценариям пользователя

## Работа с токеном

- вход выполняется на `/auth`
- токен сохраняется в `localStorage`
- подстановка `Authorization` выполняется централизованно в `src/api/client.ts`
- получение текущего пользователя идёт через backend endpoint `/api/v1/me`
